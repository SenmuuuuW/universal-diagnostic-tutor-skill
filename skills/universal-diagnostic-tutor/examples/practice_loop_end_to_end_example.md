# 练习闭环：二分查找循环不变量

这个例子展示一次跨两轮的完整闭环。Tutor 每轮只做当前需要的动作，出题后先等待，
不会同时给答案或下一题。

## 学习者

> 我刚学完二分查找的循环不变量，想检查自己是不是真的懂了。

## Tutor：出题并停下

当前只检查一个点：缩小区间后，目标为什么仍在候选区间内。

数组按升序排列，当前搜索区间是 `[left, right]`，中点是 `mid`。已知
`nums[mid] < target`，为什么下一步可以令 `left = mid + 1`？请用“被排除的
位置为什么不可能是答案”来解释，不需要写代码。

我先停在这里，等你作答。

## 学习者作答

> 因为数组有序，`mid` 左边的数都不大于 `nums[mid]`，所以它们也小于
> `target`；`mid` 本身也不是答案，因此目标如果存在，只可能在
> `[mid + 1, right]`。

## Tutor：批改、状态更新与推进判断

**结论：正确。**

你保留了循环不变量的关键理由：被删除的 `[left, mid]` 中每个值都小于
`target`，所以删除它们不会丢掉可能的答案。你也说明了为什么是 `mid + 1`，
而不是继续包含 `mid`。

- 错误类型：未发现概念或边界错误。
- 针对性修复：不需要；下一步应检查相反分支和边界情况。
- 学习状态：循环不变量已解释、已练习、已检查；下一步可以尝试近迁移。
- 准备度：谨慎推进。当前证据支持尝试近迁移，但还没有近迁移成功证据，
  也不把一次正确回答当成完整掌握。

下一题只增加一个陷阱：如果 `nums[mid] > target`，新的右边界应是 `mid` 还是
`mid - 1`？请说明边界选择与当前区间定义的关系。
