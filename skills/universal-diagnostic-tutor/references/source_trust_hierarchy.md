# Source Trust Hierarchy

Use this hierarchy when choosing learning resources for resource-augmented
answers.

## Highest Trust

### Official Documentation And Official Course Materials

- Official API, language, library, or standards documentation.
- Course pages, syllabi, lecture notes, or assignments from the institution
  offering the course.
- Official specifications for protocols, languages, and systems.

Use these for definitions, current behavior, and authoritative requirements.

### University Open Courseware

- Public university course pages.
- Open courseware lecture notes, assignments, and exams.
- Instructor-maintained reading lists and problem sets.

Use these for learning paths, prerequisites, and exam-style practice.

## Strong Trust

### Textbooks And Open Textbooks

- Standard textbooks named by courses.
- Open textbooks with clear authorship and licensing.
- Freely accessible textbook companion materials.

Use these for stable explanations and structured prerequisite chains. Do not
copy substantial textbook content into the answer or repo.

### Past Exams And Official Problem Sets

- Official past exams.
- Course-provided problem sets.
- Official answer keys or rubrics when available.

Use these to identify tested concepts, common patterns, and practice direction.
Do not turn them into answer dumps.

### Peer-Reviewed Or Authoritative Technical Sources

- Research papers for advanced AI/ML, systems, cryptography, or signal
  processing topics.
- Standards bodies, RFCs, or official technical reports.

Use these when course-level sources are not enough or when the topic is a
technical standard or research concept.

## Useful But Lower Priority

### Reputable Public Courses And Tutorials

- Well-known educational platforms with clear authorship.
- Maintained tutorials from reputable organizations.
- Instructor or researcher blogs with citations and careful reasoning.

Use these for intuition, examples, and alternate explanations after stronger
sources have been considered.

## Red Flags

Avoid or down-rank sources with:

- SEO content-farm style writing.
- Unsourced claims or unclear authorship.
- Copied answer-site content.
- No teaching reasoning, only final answers.
- Outdated course pages for fast-changing technical topics.
- Broken links or missing context.
- Hallucinated citations or unverifiable names.
- Aggregated notes with no original source.

## Practical Source Notes

- Prefer two or three strong sources over many weak links.
- For current software behavior, official docs outrank old tutorials.
- For mathematical foundations, stable textbooks and university notes are often
  enough.
- For exams, official course problem sets outrank random answer sites.
- If no trustworthy source is found, say so and teach from foundations.
