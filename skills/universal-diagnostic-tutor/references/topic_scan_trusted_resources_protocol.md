# Topic Scan And Trusted Resources Protocol

Use this protocol when a substantial STEM / AI-CS learning question would
benefit from a quick topic orientation and, when useful, trustworthy resources.

Topic scan is not a resource list. It is a compact way to orient the learner
before choosing the next best teaching step.

## Topic Scan Shape

Usually include one short line with:

- **Subject**
- **Course module**
- **Core concept**
- **Likely prerequisite**

Example:

```text
Topic scan: 线性代数 -> 向量 -> 平行关系 / 标量倍数。前置概念是向量分量和同一个标量倍数。
```

Use this naturally. Do not force a visible label when a sentence is clearer.

## When To Add Resources

Recommend or search for resources only when:

- the user asks for resources
- the learner is self-studying
- the learner is preparing for an exam
- the user asks for a plan
- the topic is broad and needs structured learning
- the current source would improve verification, practice, or misconception
  repair

Do not turn every tutoring answer into a resource section.

## Trusted Resource Priority

Prioritize:

1. university open courses or official course pages
2. official documentation for programming, tools, APIs, systems, or standards
3. well-known textbooks or open textbooks
4. reputable public course materials and problem sets
5. high-quality explanatory resources when they fit the learner's level

Never fabricate sources, course pages, exams, authors, links, or page numbers.

## If Search Is Available

- Search for current official or authoritative pages when resources matter.
- Prefer exact pages, lecture notes, assignment pages, docs sections, or
  problem-set pages over broad homepages.
- Cite or list sources clearly.
- Explain each source's role: intuition, formal definition, practice, exam
  pattern, implementation reference, or verification.

## If Search Is Not Available

Say resources are unverified suggestions by type or name. Example:

```text
I can suggest source types, but I cannot verify current links here. Look for a university linear algebra course section on vectors plus a visual explanation of transformations.
```

## Teaching Integration

Resources should support the current learning step:

- Teach the concept first.
- Mention one or two resource directions only if they help.
- Avoid dumping links.
- Avoid replacing the tutor's explanation with a source summary.
- For broad plans, pair each resource type with the gap it repairs.

## Mistakes To Avoid

- Long topic taxonomy before helping the learner.
- Recommending advanced resources before checking prerequisites.
- Listing five links when one teaching step is needed.
- Citing sources that were not actually checked.
- Treating source packs as exhaustive or as a substitute for teaching.
