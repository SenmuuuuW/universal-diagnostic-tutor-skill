# Programming And CS Foundations Source Pack

Use these sources for programming fundamentals, object-oriented thinking, data
structures, algorithms, software construction, and CS prerequisite repair. They
are starting points for teaching paths, not replacements for diagnosis.

## Sources

### Python Official Tutorial

* Link: https://docs.python.org/3/tutorial/
* Type: official documentation
* Best for: Python syntax, data structures, functions, modules, classes, and standard behavior.
* Use when: a learner needs an authoritative explanation of Python behavior or language basics.
* Strength: Official and current reference for Python semantics.
* Caution: Documentation can be dense; translate it into learner-friendly examples.
* Related subjects: programming fundamentals, scripting, debugging, data structures.
* Suggested citation label: Python Official Tutorial

### Harvard CS50x

* Link: https://cs50.harvard.edu/x/
* Type: university open course
* Best for: broad introductory CS, C, Python, memory, algorithms, data structures, and web basics.
* Use when: a learner needs a full CS on-ramp or asks what to study before university CS.
* Strength: Structured, beginner-friendly, and practice-oriented.
* Caution: It is broad; route learners to specific weeks or problem sets rather than the homepage when possible.
* Related subjects: programming fundamentals, computer science introduction, C, Python, algorithms.
* Suggested citation label: Harvard CS50x

### MIT OCW 6.100L Introduction To CS And Programming Using Python

* Link: https://ocw.mit.edu/courses/6-100l-introduction-to-cs-and-programming-using-python-fall-2022/
* Type: university open course
* Best for: computational thinking, Python programming, testing ideas, and problem decomposition.
* Use when: a learner wants a university-level programming foundation with Python.
* Strength: Good bridge from beginner programming to scientific and engineering computation.
* Caution: Check whether the learner first needs basic syntax or debugging help.
* Related subjects: Python, programming fundamentals, computational thinking, engineering computation.
* Suggested citation label: MIT OCW 6.100L Python Programming

### MIT OCW 6.006 Introduction To Algorithms

* Link: https://ocw.mit.edu/courses/6-006-introduction-to-algorithms-spring-2020/
* Type: university open course
* Best for: algorithmic complexity, sorting, hashing, trees, graphs, dynamic programming, and problem-set practice.
* Use when: a learner needs university-level algorithms structure or exam-style problem patterns.
* Strength: Strong lectures, recitations, assignments, and quizzes.
* Caution: Learners may need data structure basics before algorithm analysis.
* Related subjects: algorithms, data structures, exam prep, proof and complexity.
* Suggested citation label: MIT OCW 6.006 Introduction To Algorithms

### Princeton Algorithms

* Link: https://algs4.cs.princeton.edu/home/
* Type: university open course
* Best for: data structures, sorting, searching, graphs, strings, and algorithm implementation.
* Use when: a learner benefits from visual explanations and implementation-oriented algorithms.
* Strength: Clear organization with textbook-style explanations and code-oriented examples.
* Caution: Java examples may need translation for Python or C++ learners.
* Related subjects: algorithms, data structures, Java, graph algorithms.
* Suggested citation label: Princeton Algorithms

### UC Berkeley CS61A

* Link: https://cs61a.org/
* Type: university open course
* Best for: abstraction, functions, recursion, interpreters, and programming language ideas.
* Use when: a learner struggles with recursion, higher-order functions, or mental models of evaluation.
* Strength: Excellent for conceptual programming thinking beyond syntax.
* Caution: Course pages change by term; cite specific current pages when using them.
* Related subjects: programming fundamentals, functional programming, recursion, interpreters.
* Suggested citation label: UC Berkeley CS61A

### UC Berkeley CS61B

* Link: https://sp25.datastructur.es/
* Type: university open course
* Best for: data structures, object-oriented programming, testing, implementation tradeoffs, and project-style practice.
* Use when: a learner asks how lists, trees, maps, priority queues, or graphs behave in code.
* Strength: Strong bridge between abstract data types and real implementations.
* Caution: Some pages are term-specific; verify exact lecture, lab, or assignment links before citing.
* Related subjects: data structures, OOP, Java, software design, algorithms.
* Suggested citation label: UC Berkeley CS61B

### MIT OCW 6.005 Software Construction

* Link: https://ocw.mit.edu/courses/6-005-software-construction-spring-2016/
* Type: university open course
* Best for: specifications, testing, mutability, abstraction, equality, design, and maintainable code.
* Use when: a coding question is about reliability, design choices, or why a program structure fails.
* Strength: Good source for moving beyond "code that works" toward robust software.
* Caution: Use modern language documentation too when current API behavior matters.
* Related subjects: software engineering, testing, object-oriented programming, design.
* Suggested citation label: MIT OCW 6.005 Software Construction
