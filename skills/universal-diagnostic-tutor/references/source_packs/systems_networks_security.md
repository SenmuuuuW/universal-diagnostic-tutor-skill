# Systems, Networks, And Security Source Pack

Use these sources for computer organization, architecture, operating systems,
networking, databases, cryptography, and security fundamentals. Prefer official
or university sources for definitions, protocols, assignments, and exam-style
patterns.

## Sources

### Operating Systems: Three Easy Pieces

* Link: https://pages.cs.wisc.edu/~remzi/OSTEP/
* Type: open textbook
* Best for: processes, virtualization, concurrency, memory, file systems, and OS design concepts.
* Use when: a learner needs conceptual OS foundations such as virtual memory, scheduling, or locks.
* Strength: Open, readable, and designed for teaching core OS ideas.
* Caution: Use course or official architecture docs when the question depends on a specific system.
* Related subjects: operating systems, computer systems, concurrency, memory.
* Suggested citation label: OSTEP

### MIT 6.S081 Operating System Engineering

* Link: https://pdos.csail.mit.edu/6.S081/2024/
* Type: university open course
* Best for: operating systems labs, xv6, system calls, virtual memory, file systems, and concurrency.
* Use when: a learner wants systems-building context or problem-set style OS practice.
* Strength: Strong lab sequence that connects OS concepts to implementation.
* Caution: Lab details are demanding; diagnose C, pointers, and architecture readiness first.
* Related subjects: operating systems, C, systems programming, computer architecture.
* Suggested citation label: MIT 6.S081 Operating System Engineering

### CS:APP / CMU 15-213 Course Materials

* Link: https://www.cs.cmu.edu/~./213/
* Type: university open course
* Best for: machine-level code, memory hierarchy, linking, exceptional control flow, caching, and systems programming.
* Use when: a learner asks how code maps to hardware or why performance and memory behavior matter.
* Strength: Excellent systems bridge between programming, architecture, and OS ideas.
* Caution: Use only public course material; do not copy textbook content into answers or the repo.
* Related subjects: computer systems, architecture, C, operating systems, performance.
* Suggested citation label: CMU 15-213 Computer Systems

### UC Berkeley CS61C

* Link: https://cs61c.org/
* Type: university open course
* Best for: computer architecture, RISC-V, number representation, memory, caches, and parallelism.
* Use when: a learner needs the hardware layer beneath operating systems, compilers, or performance.
* Strength: Strong undergraduate bridge from C to machine organization.
* Caution: Term pages change; verify specific lecture or assignment links before citing.
* Related subjects: computer organization, architecture, systems programming, performance.
* Suggested citation label: UC Berkeley CS61C

### Stanford CS144 Introduction To Computer Networking

* Link: https://cs144.github.io/
* Type: university open course
* Best for: Internet architecture, layering, reliability, transport, routing, and network applications.
* Use when: a learner asks why protocols exist or how data moves across networks.
* Strength: Good source for conceptual networking plus implementation-oriented assignments.
* Caution: Prefer specific lecture, lab, or assignment pages when available.
* Related subjects: computer networks, distributed systems, operating systems, security.
* Suggested citation label: Stanford CS144 Computer Networking

### RFC Editor

* Link: https://www.rfc-editor.org/
* Type: standard
* Best for: Internet protocol specifications and authoritative protocol details.
* Use when: the question depends on exact protocol behavior or standard terminology.
* Strength: Primary source for RFC-defined protocols.
* Caution: RFCs are not beginner tutorials; translate specs into teaching language.
* Related subjects: networking, security, distributed systems, Internet protocols.
* Suggested citation label: RFC Editor

### OWASP Top 10

* Link: https://owasp.org/www-project-top-ten/
* Type: authoritative technical reference
* Best for: common web application security risks and threat categories.
* Use when: a learner asks about web security risks, secure coding, or vulnerability categories.
* Strength: Widely recognized security education resource.
* Caution: Keep security guidance educational and defensive; avoid exploit instructions.
* Related subjects: web security, software engineering, risk analysis, network security.
* Suggested citation label: OWASP Top 10

### CMU 15-445/645 Database Systems

* Link: https://15445.courses.cs.cmu.edu/fall2024/
* Type: university open course
* Best for: database internals, storage, indexing, query execution, transactions, and concurrency control.
* Use when: a learner asks how databases work under the hood.
* Strength: Strong implementation-oriented database systems course.
* Caution: Advanced; diagnose data structures, OS, and C++ readiness for project-style questions.
* Related subjects: databases, systems, storage, query processing, transactions.
* Suggested citation label: CMU 15-445 Database Systems

### PostgreSQL Tutorial

* Link: https://www.postgresql.org/docs/current/tutorial.html
* Type: official documentation
* Best for: SQL basics, relational concepts, queries, joins, aggregation, and database use.
* Use when: a learner needs official practical database behavior rather than database internals.
* Strength: Current official documentation from a major open-source database.
* Caution: For conceptual database theory, pair with a course or textbook source.
* Related subjects: databases, SQL, data modeling, software engineering.
* Suggested citation label: PostgreSQL Tutorial
