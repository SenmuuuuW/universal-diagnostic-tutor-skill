# Networks From Zero Source Pack

Use this pack when a learner is new to computer networks. Start with practical
mental models, then move to layered models, TCP/IP, routing, reliability,
DNS/HTTP, and RFCs only when the learner is ready.

## Sources

### MDN How Does The Internet Work

* Link: https://developer.mozilla.org/en-US/docs/Learn/Common_questions/Web_mechanics/How_does_the_Internet_work
* Type: tutorial
* Best for: beginner-friendly explanation of clients, servers, packets, routers, and the Internet as a system.
* Use when: a learner is starting from zero and needs the first mental model.
* Strength: Accessible, practical, and web-oriented.
* Caution: Tutorial-level; pair with deeper network sources for university study.
* Related subjects: computer networks, web basics, HTTP, DNS.
* Suggested citation label: MDN How The Internet Works

### MDN HTTP Overview

* Link: https://developer.mozilla.org/en-US/docs/Web/HTTP/Guides/Overview
* Type: official documentation
* Best for: HTTP request/response basics, clients, servers, headers, and web communication.
* Use when: a learner asks how browsers and servers communicate.
* Strength: Clear official documentation for web protocol concepts.
* Caution: HTTP is one layer of networking; connect it back to TCP/IP and DNS.
* Related subjects: HTTP, web development, networks, client-server systems.
* Suggested citation label: MDN HTTP Overview

### Computer Networking: Principles, Protocols And Practice

* Link: https://www.computer-networking.info/
* Type: open textbook
* Best for: layered networking, IP, routing, transport, reliability, application protocols, and network design concepts.
* Use when: a learner wants a structured open textbook path after beginner intuition.
* Strength: Open networking textbook designed for teaching.
* Caution: Choose specific chapters when possible rather than citing the whole site.
* Related subjects: computer networks, TCP/IP, routing, transport protocols.
* Suggested citation label: Computer Networking: Principles, Protocols And Practice

### Kurose/Ross Wireshark Labs

* Link: https://gaia.cs.umass.edu/kurose_ross/wireshark.php
* Type: problem set
* Best for: practical observation of packets, protocols, DNS, HTTP, TCP, UDP, IP, and routing behavior.
* Use when: a learner needs hands-on network intuition without guessing.
* Strength: Well-known public labs tied to a standard networking learning path.
* Caution: Use labs for authorized local learning only; avoid privacy-invasive packet capture advice.
* Related subjects: computer networks, protocols, labs, packet analysis.
* Suggested citation label: Kurose/Ross Wireshark Labs

### Stanford CS144 Introduction To Computer Networking

* Link: https://cs144.github.io/
* Type: university open course
* Best for: layering, reliability, transport, routing, Internet architecture, and implementation-oriented networking.
* Use when: a learner is ready for university-level networking after basic models.
* Strength: Strong bridge from concepts to implementation.
* Caution: Advanced for true beginners; start with MDN or open textbook material first.
* Related subjects: computer networks, systems, protocols, distributed systems.
* Suggested citation label: Stanford CS144 Networking

### RFC Editor

* Link: https://www.rfc-editor.org/
* Type: standard
* Best for: authoritative Internet protocol specifications.
* Use when: a specific protocol definition or standard term matters.
* Strength: Primary source for RFC-defined protocol behavior.
* Caution: Not a beginner source; translate specs into teaching language.
* Related subjects: networking, Internet standards, security, protocols.
* Suggested citation label: RFC Editor
