# Cryptography And Security Addendum

Use this addendum for modern cryptography, number theory connections, applied
cryptography, network security, web security, and security engineering basics.
Keep security explanations educational, defensive, and bounded.

## Safety Boundary

- Do not provide harmful cyber instructions.
- Keep security explanations educational and defensive.
- Avoid exploit walkthroughs unless the context is clearly safe, authorized,
  and focused on prevention or understanding.
- Prefer concepts, threat models, protocol reasoning, and defensive practice.

## Sources

### Stanford CS255 Introduction To Cryptography

* Link: https://crypto.stanford.edu/~dabo/cs255/
* Type: university open course
* Best for: symmetric encryption, public-key cryptography, signatures, key exchange, and security definitions.
* Use when: a learner asks why cryptography needs math or how modern crypto is structured.
* Strength: Authoritative university cryptography course.
* Caution: Advanced; diagnose modular arithmetic, probability, and proof readiness first.
* Related subjects: cryptography, number theory, security, protocols.
* Suggested citation label: Stanford CS255 Cryptography

### A Graduate Course In Applied Cryptography

* Link: https://toc.cryptobook.us/
* Type: open textbook
* Best for: modern applied cryptography, formal security goals, encryption, authentication, public-key methods, and protocols.
* Use when: a learner wants a rigorous but public crypto reference.
* Strength: Open textbook by recognized cryptography researchers.
* Caution: Graduate-level; use simpler intuition first for beginners.
* Related subjects: applied cryptography, protocols, number theory, security engineering.
* Suggested citation label: Applied Cryptography Book

### NIST FIPS 197 Advanced Encryption Standard

* Link: https://csrc.nist.gov/pubs/fips/197/final
* Type: standard
* Best for: authoritative AES terminology and standard-level reference.
* Use when: the question depends on the official definition of AES.
* Strength: Primary standards source.
* Caution: Standards are not tutorials; explain concepts before sending a learner to them.
* Related subjects: symmetric cryptography, standards, security engineering.
* Suggested citation label: NIST FIPS 197 AES

### RFC 8446 The Transport Layer Security Protocol Version 1.3

* Link: https://www.rfc-editor.org/rfc/rfc8446
* Type: standard
* Best for: authoritative TLS 1.3 protocol behavior and terminology.
* Use when: a learner asks how modern secure web transport is structured.
* Strength: Primary protocol specification.
* Caution: Too detailed for beginners; use only after explaining threat models, keys, certificates, and handshakes.
* Related subjects: applied cryptography, network security, web security, protocols.
* Suggested citation label: RFC 8446 TLS 1.3

### MIT OCW 6.857 Network And Computer Security

* Link: https://ocw.mit.edu/courses/6-857-network-and-computer-security-spring-2014/
* Type: university open course
* Best for: security protocols, cryptographic applications, network security, and security reasoning.
* Use when: a learner needs university-level security concepts tied to systems and networks.
* Strength: Official MIT OCW security course.
* Caution: Keep applications defensive and avoid turning examples into exploit instructions.
* Related subjects: network security, cryptography, systems security, protocols.
* Suggested citation label: MIT OCW 6.857 Security

### OWASP Web Security Testing Guide

* Link: https://owasp.org/www-project-web-security-testing-guide/
* Type: authoritative technical reference
* Best for: defensive web security testing concepts, risk categories, and structured assessment language.
* Use when: a learner asks how web security is evaluated in practice.
* Strength: Widely recognized defensive security resource.
* Caution: Use for authorized, defensive learning only; avoid procedural exploit walkthroughs.
* Related subjects: web security, software engineering, threat modeling.
* Suggested citation label: OWASP WSTG
