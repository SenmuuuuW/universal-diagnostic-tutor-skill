# VR And Multimedia Addendum

Use this addendum for virtual reality basics, graphics/math prerequisites,
multimedia systems, image/video/audio processing, and HCI connections. Diagnose
whether the learner needs math, rendering, perception, media encoding, or
interface-design foundations first.

## Sources

### Khronos OpenXR Specification

* Link: https://registry.khronos.org/OpenXR/specs/1.0/html/xrspec.html
* Type: standard
* Best for: authoritative OpenXR concepts, runtime/application model, sessions, spaces, input, and rendering structure.
* Use when: a learner asks how VR/AR runtime standards organize immersive applications.
* Strength: Primary standard for OpenXR.
* Caution: Standards are dense; use only after giving a beginner mental model. Command-line verification may fail in some environments; verify before citing exact sections.
* Related subjects: virtual reality, augmented reality, graphics APIs, standards.
* Suggested citation label: Khronos OpenXR Specification

### MDN WebXR Device API

* Link: https://developer.mozilla.org/en-US/docs/Web/API/WebXR_Device_API
* Type: official documentation
* Best for: browser-based XR concepts, sessions, reference spaces, devices, and web implementation vocabulary.
* Use when: a learner asks how VR/AR works in web environments.
* Strength: Accessible documentation with practical API framing.
* Caution: API docs should support concepts; do not let API details replace teaching.
* Related subjects: WebXR, VR, AR, web development, HCI.
* Suggested citation label: MDN WebXR Device API

### Immersive Web WebXR Samples

* Link: https://immersive-web.github.io/webxr-samples/
* Type: tutorial
* Best for: practical WebXR examples showing device interaction and immersive scene patterns.
* Use when: a learner needs to connect WebXR terms to visible behavior.
* Strength: Public examples maintained by the immersive web ecosystem.
* Caution: Treat as examples, not formal theory or a full VR curriculum.
* Related subjects: WebXR, VR implementation, graphics, interaction.
* Suggested citation label: Immersive Web WebXR Samples

### MIT OCW 6.837 Computer Graphics

* Link: https://ocw.mit.edu/courses/6-837-computer-graphics-fall-2012/
* Type: university open course
* Best for: transformations, cameras, rendering, ray tracing, shading, geometry, and animation foundations.
* Use when: VR or multimedia confusion comes from graphics/math prerequisites.
* Strength: Strong university graphics foundation.
* Caution: Some toolchains are dated; use for concepts rather than modern API details.
* Related subjects: computer graphics, VR, rendering, linear algebra.
* Suggested citation label: MIT OCW 6.837 Computer Graphics

### Stanford CS131 Computer Vision

* Link: https://stanford.edu/class/cs131/
* Type: university open course
* Best for: image formation, filtering, features, recognition, and computer vision foundations.
* Use when: a learner asks how image processing or perception connects to multimedia and AI.
* Strength: Strong public course for vision and image processing basics.
* Caution: Verify exact assignment or lecture pages when citing specific topics.
* Related subjects: computer vision, image processing, pattern recognition, AI.
* Suggested citation label: Stanford CS131 Computer Vision

### MDN Web Audio API

* Link: https://developer.mozilla.org/en-US/docs/Web/API/Web_Audio_API
* Type: official documentation
* Best for: audio graphs, nodes, signal flow, processing, and browser-based audio concepts.
* Use when: a learner asks how audio becomes programmable multimedia data.
* Strength: Official documentation with practical conceptual vocabulary.
* Caution: Pair with signals sources when explaining sampling, filters, or frequency content.
* Related subjects: audio processing, multimedia, web development, signals.
* Suggested citation label: MDN Web Audio API

### FFmpeg Documentation

* Link: https://ffmpeg.org/documentation.html
* Type: official documentation
* Best for: multimedia containers, encoding/decoding tools, filters, and practical media-processing vocabulary.
* Use when: a learner asks how video/audio processing tools fit into multimedia workflows.
* Strength: Official documentation for a major multimedia toolchain.
* Caution: Tool docs are not an introductory textbook; teach codecs, containers, and sampling first.
* Related subjects: multimedia, video processing, audio processing, software tools.
* Suggested citation label: FFmpeg Documentation
