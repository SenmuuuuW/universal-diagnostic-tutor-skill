# AI, Machine Learning, And Data Source Pack

Use these sources for AI, machine learning, neural networks, data mining,
information retrieval, search engines, embeddings, and framework-backed study.
Always diagnose math, programming, and conceptual prerequisites before
recommending resources.

## Sources

### Stanford CS229 Machine Learning

* Link: https://cs229.stanford.edu/
* Type: university open course
* Best for: supervised learning, generalization, optimization, probabilistic models, and core ML foundations.
* Use when: a learner wants rigorous university-level ML concepts.
* Strength: Authoritative ML course with strong mathematical framing.
* Caution: Diagnose calculus, linear algebra, probability, and programming readiness first.
* Related subjects: machine learning, optimization, probability, linear algebra.
* Suggested citation label: Stanford CS229 Machine Learning

### Stanford CS224N Natural Language Processing With Deep Learning

* Link: https://web.stanford.edu/class/cs224n/
* Type: university open course
* Best for: embeddings, neural NLP, sequence models, transformers, language models, and evaluation.
* Use when: a learner asks about embeddings, NLP, representation learning, or neural language models.
* Strength: Strong bridge between linear algebra intuition and modern NLP systems.
* Caution: Advanced; explain vectors, matrix multiplication, and probability before deep architectures.
* Related subjects: NLP, deep learning, embeddings, machine learning, AI.
* Suggested citation label: Stanford CS224N NLP

### Google Machine Learning Crash Course

* Link: https://developers.google.com/machine-learning/crash-course
* Type: tutorial
* Best for: beginner ML concepts, loss, training, features, classification, and practical intuition.
* Use when: a learner needs an accessible on-ramp before formal ML courses.
* Strength: Practical, interactive, and beginner-friendly.
* Caution: Pair with university or textbook sources for deeper theory.
* Related subjects: machine learning, data science, model training, evaluation.
* Suggested citation label: Google Machine Learning Crash Course

### Dive Into Deep Learning

* Link: https://d2l.ai/
* Type: open textbook
* Best for: neural networks, deep learning implementation, optimization, convolution, sequence models, and attention.
* Use when: a learner wants an open textbook that connects theory with code.
* Strength: Broad deep learning coverage with executable examples.
* Caution: Framework details can change; verify current library behavior with official docs.
* Related subjects: deep learning, neural networks, machine learning, AI implementation.
* Suggested citation label: Dive Into Deep Learning

### scikit-learn User Guide

* Link: https://scikit-learn.org/stable/user_guide.html
* Type: official documentation
* Best for: classical ML models, preprocessing, pipelines, evaluation, and implementation behavior.
* Use when: a learner asks how common ML algorithms are used in Python practice.
* Strength: Official source for scikit-learn concepts and API behavior.
* Caution: Documentation explains tools; still teach the statistical idea behind the tool.
* Related subjects: machine learning, Python, data science, model evaluation.
* Suggested citation label: scikit-learn User Guide

### PyTorch Tutorials

* Link: https://docs.pytorch.org/tutorials/
* Type: official documentation
* Best for: tensor operations, neural network training loops, autograd, datasets, and model implementation.
* Use when: a learner asks how deep learning concepts become runnable code in PyTorch.
* Strength: Official framework tutorials and current implementation patterns.
* Caution: Separate framework mechanics from the underlying ML concept.
* Related subjects: deep learning, tensors, neural networks, Python, AI implementation.
* Suggested citation label: PyTorch Tutorials

### TensorFlow Tutorials

* Link: https://www.tensorflow.org/tutorials
* Type: official documentation
* Best for: TensorFlow/Keras workflows, model training, computer vision, NLP, and deployment-oriented examples.
* Use when: a learner is using TensorFlow or comparing ML framework workflows.
* Strength: Official source for TensorFlow behavior and examples.
* Caution: Avoid mixing TensorFlow and PyTorch explanations unless the learner asks to compare.
* Related subjects: deep learning, TensorFlow, model training, AI implementation.
* Suggested citation label: TensorFlow Tutorials

### Stanford CS231n Convolutional Neural Networks For Visual Recognition

* Link: https://cs231n.github.io/
* Type: university open course
* Best for: image classification, convolutional networks, backpropagation, optimization, and vision models.
* Use when: a learner asks about image processing, computer vision, neural networks, or representation learning.
* Strength: Clear notes connecting math, implementation, and visual intuition.
* Caution: Some details may reflect a specific course version; verify current model practice when needed.
* Related subjects: computer vision, neural networks, image processing, ML.
* Suggested citation label: Stanford CS231n

### Stanford CS276 Information Retrieval And Web Search

* Link: https://web.stanford.edu/class/cs276/
* Type: university open course
* Best for: search engines, indexing, ranking, retrieval models, evaluation, and web search foundations.
* Use when: a learner asks how search engines connect to data mining or NLP.
* Strength: Focused university course on information retrieval and search.
* Caution: For textbook-style depth, pair with the IR book source below.
* Related subjects: search engines, information retrieval, data mining, NLP.
* Suggested citation label: Stanford CS276 Information Retrieval

### Introduction To Information Retrieval

* Link: https://nlp.stanford.edu/IR-book/
* Type: open textbook
* Best for: inverted indexes, retrieval models, ranking, evaluation, clustering, and web search basics.
* Use when: a learner needs structured reading for search engines or information retrieval.
* Strength: Open textbook with clear IR foundations.
* Caution: Do not copy long passages; summarize and cite specific chapters when used.
* Related subjects: information retrieval, search engines, NLP, data mining.
* Suggested citation label: Introduction To Information Retrieval
