# 群友使用指南：诊断式理科学习导师

这是一份可以直接发到群里的简明说明。它面向普通用户、学生和自学者，不要求你懂 GitHub、Codex 或 Claude Code。

如果你需要从“点进 GitHub 后先看哪里”开始的完整教程，请看 `USER_GUIDE.md`。这份文件保持短一点，方便直接发群。

## 这是什么？

这是一个诊断式学习导师提示词 / Skill。它的重点不是“立刻把答案给你”，而是先判断：

- 这题属于哪个学科和知识点？
- 你卡在概念、符号、方法、步骤，还是迁移？
- 现在最值得讲的一小步是什么？

它适合用来学懂题、补基础、分析错因、做备考复习和跨 chat 继续学习。

## 谁适合用？

- 高中 / 大学理科学习者
- 大学高数、线代、概率统计、离散数学、大学物理学习者
- CS / AI-CS 学习者
- 数据结构、算法、计组、操作系统、计算机网络复习用户
- 考研数学 / CS 专业课复习用户
- 想知道“我到底哪里不会”的人

它不是答案库、押题工具、作弊工具，也不会保证提分。

## 普通 AI 怎么用？

如果你用的是普通 ChatGPT、Gemini、DeepSeek、豆包、Kimi、Qwen 这类聊天窗口：

1. 打开这个文件：`platforms/generic-chat/TUTOR_LITE_PROMPT.md`
2. 把里面的 Lite Prompt 复制到聊天窗口。
3. 再发你的题目或问题。

如果你不会 GitHub，先打开 `platforms/generic-chat/TUTOR_LITE_PROMPT.md`，复制到你常用的 AI chat 里。普通用户不需要先 clone repo。

普通聊天不会自动读取整个仓库，所以它是 Lite 版，但核心风格仍然是：先诊断，再教学。

如果你问的是很大的学习目标，比如“我想学机器学习”“我想补线代”或“我想准备考试”，Tutor 会先确认你到底想学什么，再给一个小知识地图和第一步。这样它不是直接乱讲，而是先确定方向。

如果你用的是 Codex，可以选择 `tutor-study-plan`、`tutor-exam-track`、
`tutor-state-card` 这些入口；如果是普通聊天 AI，直接用 Lite Prompt 里的文字指令即可。

## 豆包 / Coze 风格 Bot 怎么用？

如果你在做豆包 / Coze 风格智能体，可以使用：

```text
platforms/coze-doubao/BOT_PROMPT.md
```

把这个 prompt 放进 bot 的角色说明或系统提示里。不同平台的知识库和记忆机制不同，不要假设它会自动拥有跨聊天记忆。

## 第一道题怎么问？

可以直接复制这些问法：

```text
我是零基础，请先判断这题属于哪个知识点，不要直接给完整答案。
```

```text
我会一点但卡在第一步，请先找我缺的概念。
```

```text
我做错了，这是我的思路，帮我分析错在哪里。
```

```text
/exam-track 我准备考试，这个知识点总做错，帮我诊断和安排复习。
```

```text
/visualize 这个概念我看文字不懂，能不能画成简单图或用表格讲？
```

## 怎么继续学习？

如果这次没学完，直接说：

```text
帮我生成一个下次继续用的 Learning State Card。
```

下次开新 chat 时，把那张卡粘贴进去，再说：

```text
根据这张卡继续，不要从头讲。
```

注意：这不是隐藏记忆。它只是你自己复制保存的一张学习状态卡。

## 怎么反馈问题？

如果你觉得回答不好，反馈时最好带上：

- 学科 / 主题
- 你的原始提问
- AI 的错误回答
- 你期待它怎么做
- 问题类型：太快、太慢、直接给答案太早、没有诊断、资料太多、资料太少、图示没帮助

这样维护时可以把问题转成小的测试案例，而不是盲目加功能。

## 一个小提醒

最有效的用法不是“给我答案”，而是：

```text
先判断我卡在哪里，只讲下一小步，问我一个检查题后停。
```

这句话很朴素，但它正是这个导师最有用的地方。
